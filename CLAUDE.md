## Архитектура — прочитать перед любыми изменениями

**Чистый SPA, без бэкенда, без базы данных.**

```
src/
├── App.tsx       — ВСЕ экраны и логика. Менять экраны — только здесь.
├── constants.ts  — ВСЕ данные и тексты. Менять контент — только здесь.
├── main.tsx      — Точка входа (StrictMode + createRoot). Не трогать.
└── index.css     — Глобальные стили Tailwind. Не трогать без причины.
```

**Поток экранов (строго последовательно):**
```
intro → welcome → explanation → action → thankyou
```
Переключение через `useState<Screen>` — никакого роутера нет.

**Постоянные элементы (на всех экранах):**
- `ProgressBar` — 3 сегмента, скрыт на `intro` и `thankyou`
- `BottomNav` — кнопки «Начать / Написать / Поделиться»
- `OfferModal` — показывается один раз через 1.5 сек, флаг `offer_shown_v1` в localStorage

## Telegram SDK — как работает

В `index.html` подключён `https://telegram.org/js/telegram-web-app.js`.

В браузере `window.Telegram` = `undefined` — это нормально, код это обрабатывает везде через `if (window.Telegram?.WebApp)`.

| Метод | Где используется | Браузер fallback |
|-------|-----------------|-----------------|
| `tg.ready()` / `tg.expand()` | App useEffect | Игнорируется |
| `tg.setHeaderColor('#f5f5f0')` | App useEffect | Игнорируется |
| `tg.openTelegramLink(url)` | Все CTA-кнопки | `window.open(url, '_blank')` |
| `tg.initDataUnsafe.user.first_name` | Персонализация Intro | Пустая строка → заголовок без имени |

**Не реализовано (задачи в PLAN.md):**
- `tg.BackButton` — системная кнопка «Назад» Telegram сейчас закрывает Mini App вместо навигации назад
- `tg.HapticFeedback` — вибрация при выборе симптома и CTA

## Как запустить локально

```bash
cd mvp_extracted
npm install
cp .env.example .env   # значения по умолчанию подходят, редактировать не нужно
npm run dev            # → http://localhost:3000
```

**.env переменные:**
```
TELEGRAM_BOT_TOKEN=...  # не используется во фронтенде (SPA без бэкенда), хранится в Vercel
GEMINI_API_KEY=...      # зарезервировано, не используется
APP_URL=http://localhost:3000
```

**Полезные команды:**
```bash
npm run lint    # проверка TypeScript (должна завершиться без вывода)
npm run build   # продакшн-сборка в dist/
```

## Принципы работы с кодом

- Менять только то, о чём явно попросили — не улучшать попутно
- Перед созданием нового файла — объяснить что и зачем
- Все тексты и данные — только в `constants.ts`, не хардкодить в `App.tsx`
- Стиль текста в приложении: простой, дружеский, как живой человек — не «осуществить взаимодействие», а «написать»

## Контекст бизнеса
