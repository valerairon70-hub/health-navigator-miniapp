# Техническое задание: Навигатор самочувствия
> Telegram Mini App · Версия 1.0 · Март 2026

---

## 1. О ПРОЕКТЕ

### Цель приложения
Помочь человеку с размытыми жалобами (усталость, плохой сон, лишний вес, стресс, ЖКТ) за 3 шага понять, что стоит за его симптомами — и мягко подвести к разговору с wellness-консультантом.

Приложение не ставит диагнозов. Оно объясняет связи между симптомами и системами организма простым языком, формирует доверие и создаёт естественный запрос на консультацию.

### Целевая аудитория

**Пользователь (клиент):**
- Женщина или мужчина 25–50 лет
- Жалобы: «всегда устаю», «не могу похудеть», «плохо сплю», «постоянный стресс»
- Не ищет диагноз — ищет объяснение и понимание
- Попадает в приложение через Telegram по ссылке от знакомого или специалиста
- Скептичен: «что мне хотят продать?»

**Владелец (специалист):**
- Wellness-консультант, нутрициолог, специалист по здоровью
- Использует приложение как инструмент прогрева аудитории
- Получает из навигатора тёплых лидов, готовых к разговору
- Текущий специалист: **Валерий Созанов**, Wellness-консультант

### Чего НЕТ в приложении (намеренно)
- Постановки диагнозов или медицинских рекомендаций
- Регистрации, личного кабинета, хранения данных
- Автоматических ответов — только живой специалист
- Нескольких специалистов (пока)

## 3. ДИЗАЙН-СИСТЕМА

### Цвета
| Название | Hex | Применение |
|---|---|---|
| Фон | `#f5f5f0` | Фон всего приложения — тёплый офф-вайт |
| Акцент | `#5A5A40` | Кнопки, заголовки секций, бордюры, иконки |
| Текст основной | `#1a1a1a` | Заголовки и основной текст |
| Текст вторичный | `#1a1a1a` с `opacity-60` | Подписи, пояснения |
| Белый | `#ffffff` | Карточки, блоки |
| Зелёный | `emerald-500 / emerald-700` | Бейдж «Бесплатно», индикатор срочности |

### Шрифты
- **Заголовки:** serif (Georgia-style) — `font-serif` — создаёт ощущение экспертности
- **Текст, кнопки, подписи:** system sans-serif — `font-sans` — читабельность

### Размеры и скругления
- Карточки: `rounded-2xl` / `rounded-3xl`
- Кнопки: `rounded-2xl`, padding `py-5`
- Отступ контента: `px-6`
- Максимальная ширина блока: `max-w-md` (мобильный формат)

### Анимации
- Переход между экранами: `fade + slide` через `AnimatePresence`
- Экран intro/welcome: slide up (`y: 20 → 0`)
- Экран explanation/selection: slide right (`x: 20 → 0`)
- Финальный экран action: scale (`scale: 0.95 → 1`)
- Индикатор срочности: `animate-pulse`

### Настроение
Тёплое, спокойное, доверительное. Не медицинское, не клиническое. Как разговор с умным другом, который разбирается в здоровье.

## 4. СТРУКТУРА ЭКРАНОВ

Приложение состоит из 5 экранов, следующих строго по порядку:

```
intro → welcome → explanation → selection → action
```

Прогресс отображается полосой из 3 сегментов (шаг 0 = скрыта, шаг 1–3 = активна).

### Экран 0: INTRO (Приветствие)

**Назначение:** Захватить внимание, установить ожидание, снизить барьер входа.

**URL-состояние:** `screen === 'intro'`

**Анимация входа:** `opacity: 0, y: 20` → `opacity: 1, y: 0`

#### Секция: Шапка
- Логотип: круглая иконка `#5A5A40` со Sparkles внутри
- Название: «Навигатор» — маленькое, uppercase, opacity 60%

#### Секция: Социальное доказательство + бейдж
- Бейдж «Бесплатно» — зелёный, rounded-full, uppercase
- Рядом: «[N] человек уже прошли навигатор» — мелко, opacity 50%
- ⚠️ Цифру `count` использовать ТОЛЬКО реальную. Фейк подрывает доверие.

#### Секция: Заголовок (главный крючок)
```
Почему ты устаёшь, плохо спишь или не можешь сбросить вес — и что за этим стоит
```
- Шрифт: serif, `text-3xl`, `font-light`
- Принцип: называть конкретные симптомы из жизни — пользователь должен узнать себя

#### Секция: Подзаголовок
```
Ответь на 3 вопроса и получи бесплатное объяснение своих симптомов от специалиста.
```
- Шрифт: sans, `text-base`, opacity 60%

#### Секция: Список выгод (3 пункта)
Каждый пункт — иконка-точка + текст:
1. «3 вопроса — 2 минуты»
2. «Простой язык, без медицинских терминов»
3. «Реальный специалист, не бот»

**Важно:** Список выгод = установка ожидания. Увеличивает completion rate на 25–40%.

#### Секция: CTA-кнопка
```
Узнать, что со мной происходит →
```
- Стиль: `bg-[#5A5A40]`, белый текст, `rounded-2xl`, `py-5`, full width

## 7. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Telegram WebApp SDK — обязательно
```typescript
tg.ready();              // ✓ реализовано
tg.expand();             // ✓ реализовано
tg.setHeaderColor();     // ✓ реализовано
tg.openTelegramLink();   // ✓ реализовано для CTA
```

### Telegram WebApp SDK — нужно добавить
```typescript
// BackButton — предотвращает закрытие при нажатии «назад»
tg.BackButton.show();
tg.BackButton.onClick(() => goBack());
// на экране intro: tg.BackButton.hide()

// HapticFeedback — тактильный отклик
tg.HapticFeedback.impactOccurred('medium');        // при выборе симптома
tg.HapticFeedback.notificationOccurred('success'); // при нажатии финального CTA
```

### Константы (src/constants.ts)
Все изменяемые данные вынесены в `constants.ts`. Менять только там:

| Константа | Описание | Статус |
|---|---|---|
| `SPECIALIST.name` | Имя специалиста | ✓ Валерий Созанов |
| `SPECIALIST.photo` | Путь к фото (`/specialist.png`) | ✓ установлено |
| `SPECIALIST.username` | Telegram username | ✓ valeriisozanov |
| `SPECIALIST.fact` | Факт о специалисте | Обновить на реальный |
| `SPECIALIST.testimonial` | Отзыв клиента | Заменить на реальный |
| `SPECIALIST.slotsLeft` | Осталось мест | Убрать или обновлять вручную |
| `SOCIAL_PROOF.count` | Число прошедших | Убрать до накопления реальных данных |

### Производительность и совместимость
- HTTPS обязателен (Vercel обеспечивает автоматически)
- Вёрстка mobile-first, `max-w-md` — для экрана телефона внутри Telegram
- iOS: viewport сдвигается при появлении клавиатуры — учитывать если добавлять текстовые поля
- Нельзя открыть чат без `@username` — использовать только username, не user_id
