# Руководство тестировщика: Навигатор самочувствия

> Telegram Mini App. Продакшн: https://mvpextracted.vercel.app
> Прочитай этот файл от начала до конца перед тем, как что-либо запускать.

---

## 1. Что это и как устроено

Одностраничное приложение (SPA) без бэкенда и базы данных. Весь стейт — в памяти браузера, одно исключение — `localStorage` для флага показа OfferModal.

**Технологии:** React 19 + TypeScript + Vite + Tailwind CSS 4 + Motion (framer-motion аналог)

**Файлы с логикой:**
- `src/App.tsx` — все экраны и навигация
- `src/constants.ts` — все тексты, данные специалиста, темы симптомов

**Экранов 5, строго последовательно:**
```
intro → welcome → explanation → action → thankyou
```

**Никакого роутинга нет** — переключение экранов через `useState<Screen>`.

## 2. Локальный запуск

### Требования
- Node.js 18+ (проверить: `node -v`)
- npm 9+ (проверить: `npm -v`)

### Шаги

```bash
# 1. Перейти в папку проекта
cd /путь/к/health-navigator-miniapp/mvp_extracted

# 2. Установить зависимости
npm install

# 3. Создать .env (обязательно, иначе vite не соберёт process.env)
cp .env.example .env
# Содержимое .env.example:
#   GEMINI_API_KEY=your_key_here   ← не используется, оставь любое значение
#   APP_URL=http://localhost:3000

# 4. Запустить
npm run dev
```

После запуска: **http://localhost:3000**

> Сервер запускается с флагом `--host=0.0.0.0` — доступен по локальной сети. Это важно для тестирования с телефона через Wi-Fi (см. раздел 5).

### Проверка сборки и типов

```bash
# Проверка TypeScript (без компиляции)
npm run lint

# Проверка продакшн-сборки
npm run build
# Должна завершиться без ошибок. Результат — в папке dist/
```

## 3. Архитектура: что важно знать тестировщику

### localStorage
Ключ `offer_shown_v1` — флаг того, что OfferModal уже показывался.
- Если ключа нет → модал появится через 1.5 секунды после открытия
- После закрытия модала → ключ записывается → при следующем открытии модала не будет

**Сброс для повторного тестирования:**
```
DevTools → Application → Local Storage → http://localhost:3000 → удалить offer_shown_v1
```

### Telegram SDK
В `index.html` подключён скрипт `https://telegram.org/js/telegram-web-app.js`.
В браузере `window.Telegram` будет `undefined` — это нормально.
Код это обрабатывает: везде есть проверка `if (window.Telegram?.WebApp)`.

**Что работает в браузере:**
- Вся навигация между экранами
- Все анимации
- Все тексты и изображения
- OfferModal
- Кнопка «Поделиться» — не откроет Telegram, ничего не произойдёт (это нормально)

**Что работает только в Telegram:**
- `tg.expand()` — развернуть на весь экран
- `tg.setHeaderColor('#f5f5f0')` — цвет шапки Telegram
- `tg.openTelegramLink()` — открытие чата (CTA-кнопка и кнопка шаринга)
- Персонализация заголовка по имени (`tg.initDataUnsafe.user.first_name`)

### Изображения
- `public/specialist.png` — фото специалиста. Если файл отсутствует → сломанная иконка в карточке.
- `public/icon.png` — иконка бота (не используется в UI)
- `public/welcome-banner.svg` — баннер BotFather (не используется в UI)

## 9. Известные ограничения (не баги)

Это намеренные упрощения MVP — не фиксировать как баги:

| Что | Почему это норма |
|-----|-----------------|
| Кнопка «Поделиться» не работает в браузере | Требует Telegram SDK — `tg.openTelegramLink` |
| CTA открывает новую вкладку в браузере, а не чат внутри Telegram | Fallback для браузера через `window.open` |
| Имя пользователя не подставляется в браузере | Требует `tg.initDataUnsafe` — доступен только в Telegram |
| `GEMINI_API_KEY` в `.env` не используется | Зарезервировано для будущего |
