## Принцип проектирования

**Не строить завтра то, что не нужно сегодня.**

Каждая фаза — рабочий продукт, который можно монетизировать. Переход к следующей фазе происходит только тогда, когда текущей уже недостаточно.

```
Фаза 1 (сейчас)     Фаза 2 (рост)        Фаза 3 (масштаб)
5–10 человек    →   10–50 человек    →   50+ человек
Без бэкенда         Лёгкий бэкенд        Полноценный SaaS
Ручная настройка    Полуавтомат.         Самообслуживание
```

## Решение по боту: каждый получает СВОЙ бот ✅

**Рекомендация: собственный бот у каждого дистрибьютора.**

**Почему:**
- CoralClub дистрибьюторы строят личный бренд — бот должен называться «Навигатор Анны», не «общий CoralClub бот»
- Независимость: проблема у одного дистрибьютора не затрагивает других
- Профессиональнее выглядит для конечного клиента
- Telegram не ограничивает количество ботов — каждый создаёт через @BotFather за 2 минуты

**Схема:**
```
Дистрибьютор А → @anna_health_bot  → healthnav.vercel.app?d=anna
Дистрибьютор Б → @ivan_health_bot  → healthnav.vercel.app?d=ivan
Дистрибьютор В → @olga_health_bot  → healthnav.vercel.app?d=olga
              ↑                              ↑
        Свой бот в Telegram          Одно приложение, разные конфиги
```

Одно приложение на Vercel. URL-параметр `?d=slug` определяет чей конфиг загружать.

## Фаза 1 — Статическая мультиарендность

> **Цель:** 5–10 дистрибьюторов, ручная настройка с помощью Валерия.
> **Бэкенд:** не нужен — конфиги как JSON-файлы прямо в репозитории.
> **Время разработки:** 2–3 недели.

### Как работает

```
git repo/
├── src/                        # Общий код (не трогается)
├── public/
│   └── distributors/
│       ├── anna/
│       │   └── photo.png       # Фото дистрибьютора
│       └── ivan/
│           └── photo.png
├── configs/
│   ├── _default.json           # Базовый конфиг (шаблон)
│   ├── anna.json               # Конфиг Анны
│   └── ivan.json               # Конфиг Ивана
└── src/
    └── config-loader.ts        # Загружает нужный конфиг по URL-параметру
```

### Структура конфига (configs/anna.json)

```json
{
  "slug": "anna",
  "plan": "demo",
  "specialist": {
    "name": "Анна Смирнова",
    "role": "Специалист по восстановлению здоровья",
    "photo": "/distributors/anna/photo.png",
    "fact": "5 лет практики · 150+ клиентов",
    "username": "anna_smirnova",
    "botUsername": "anna_health_bot",
    "shareText": "Прошла тест — точно описал мои симптомы. Попробуй:",
    "slotsLeft": 3
  },
  "branding": {
    "primaryColor": "#5A5A40",
    "backgroundColor": "#f5f5f0"
  },
  "socialProof": {
    "count": 0,
    "label": "человек уже разобрались"
  },
  "topics": "default",
  "testimonials": "default",
  "offer": "default"
}
```

`"topics": "default"` означает — использовать базовые темы из `_default.json`.
Позже можно заменить на массив с индивидуальными темами.

**Onboarding Валерия (новый дистрибьютор):**
1. Дистрибьютор заполняет простую форму (Google Form или Telegram)
2. Валерий (с помощью Claude Code) создаёт `configs/slug.json` и добавляет фото
3. Делает коммит → Vercel деплоит автоматически
4. Даёт дистрибьютору инструкцию по настройке бота в BotFather
5. Готово — бот работает

**Обновления:**
- Валерий делает изменения в коде → `git push` → все дистрибьюторы получают обновление автоматически
- Конфиги дистрибьюторов не затрагиваются при обновлении кода

## Фаза 2 — Лёгкий бэкенд

> **Триггер перехода:** 10+ дистрибьюторов, ручная настройка занимает слишком много времени.
> **Бэкенд:** Supabase (PostgreSQL + Auth + Storage) + Vercel Serverless Functions.
> **Цель:** дистрибьютор настраивает себя сам через форму.

### Почему Supabase

- Бесплатный tier: до 50 000 активных пользователей, 500 MB БД — хватит надолго
- Встроенная авторизация (email, Telegram Login Widget)
- Файловое хранилище для фото (заменяет папку `public/distributors/`)
- Готовое API — не нужно писать CRUD с нуля
- Разворачивается за 10 минут, управляется через UI

### Схема базы данных

```sql
-- Дистрибьюторы
distributors (
  id          uuid PRIMARY KEY,
  slug        text UNIQUE NOT NULL,        -- 'anna', 'ivan'
  email       text UNIQUE,
  tg_user_id  bigint UNIQUE,               -- Telegram user ID
  plan        text DEFAULT 'demo',         -- 'demo' | 'basic' | 'pro'
  plan_expires_at timestamptz,
  created_at  timestamptz DEFAULT now(),
  is_active   boolean DEFAULT true
)

-- Конфиги (JSON в БД, как сейчас в файлах)
distributor_configs (
  id              uuid PRIMARY KEY,
  distributor_id  uuid REFERENCES distributors,
  specialist      jsonb NOT NULL,           -- имя, фото, username и т.д.
  branding        jsonb DEFAULT '{}',       -- цвета
  social_proof    jsonb DEFAULT '{}',       -- счётчик
  topics          jsonb DEFAULT '[]',       -- темы ([] = default)
  testimonials    jsonb DEFAULT '[]',       -- отзывы ([] = default)
  offer           jsonb DEFAULT '{}',       -- модал оффера
  updated_at      timestamptz DEFAULT now()
)

-- Подписки (история)
subscriptions (
  id              uuid PRIMARY KEY,
  distributor_id  uuid REFERENCES distributors,
  plan            text,
  amount          integer,                 -- в рублях
  started_at      timestamptz,
  expires_at      timestamptz,
  payment_ref     text                     -- ссылка на платёж
)
```

### API (Vercel Serverless Functions)

```
GET  /api/config?d=anna         → возвращает конфиг дистрибьютора
POST /api/distributors/register  → регистрация нового дистрибьютора
PUT  /api/distributors/config    → обновление конфига (авторизован)
GET  /api/distributors/me        → профиль текущего дистрибьютора
POST /api/subscriptions          → создать/продлить подписку
```

### Личный кабинет дистрибьютора (Фаза 2)

Простая страница `/setup` в том же приложении:

```
/setup — форма настройки профиля
  - Имя и роль
  - Загрузка фото (→ Supabase Storage)
  - Telegram username
  - Количество мест
  - (позже) темы и отзывы

/setup/preview — предпросмотр как выглядит навигатор
```

Авторизация: Telegram Login Widget (кнопка «Войти через Telegram»).

## Фаза 3 — Полноценный SaaS

> **Триггер перехода:** 50+ дистрибьюторов, нужна автоматизация и аналитика.

- Подключение платёжной системы (Telegram Stars / YooKassa / ЮМани)
- Автоматическое выставление счётов и блокировка при просрочке
- Аналитика на дашборде: сколько прошли воронку, какие темы популярны
- Глубокая кастомизация: темы, тексты, цвета
- Реферальная программа между дистрибьюторами
- Многоязычность (украинский, английский)

## Что НЕ строим сейчас (намеренно)

| Что | Почему нет |
|-----|-----------|
| Панель администратора | На 5–10 человек — JSON-файлы + Claude Code быстрее |
| Автоматическая регистрация | Ручной onboarding лучше контролирует качество на старте |
| Платёжная система | Сначала доказать ценность, потом монетизировать |
| Аналитика на дашборде | PostHog решает задачу без собственного дашборда |
| Кастомные цвета в UI | Минимальная кастомизация на Фазе 1 |
