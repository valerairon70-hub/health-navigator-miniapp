## Часть 1: Идеальный конечный результат (ИКР)

### Суть в одном предложении

Приложение-фильтр, которое за 2–3 минуты отделяет людей с реальным запросом на восстановление здоровья от случайно зашедших — и направляет первых на консультацию к дистрибьютору.

---

### Проблема которую решает приложение

Дистрибьютор тратит время на людей, которым «просто интересно» — они задают вопросы, берут информацию и не двигаются дальше. Приложение решает это: человек сам проходит тест, сам разбирается в причинах своих симптомов, сам принимает решение записаться на консультацию. К дистрибьютору приходит уже осознанный, мотивированный человек.

**Ключевое:** никакого упоминания компании, продуктов, цен и MLM. Это не продающий каталог — это инструмент осознания. Человек должен сам захотеть разобраться в своём здоровье.

---

### Путь пользователя: точка А → точка Б

```
Симптом (точка А)
    ↓
Понимает причины своего состояния
    ↓
Записывается на консультацию к дистрибьютору
    ↓
Получает персональную программу восстановления
    ↓
Проходит путь под сопровождением дистрибьютора
    ↓
Желаемый результат по здоровью (точка Б)
```

---

### Роль приложения в этой цепочке

Приложение выполняет **только первые два шага** — и это намеренно:

1. Помогает пользователю осознать причины своих симптомов простым языком
2. Формирует желание разобраться глубже — через консультацию

Всё остальное — программа, продукты, сопровождение — это живая работа дистрибьютора. Приложение не продаёт, не объясняет продукт, не называет компанию. Оно фильтрует и направляет.

---

### Кто использует приложение

**Дистрибьютор (владелец инструмента):**
- Настраивает приложение под себя один раз (имя, фото, контакт, темы)
- Распространяет ссылку на бота через Telegram, Instagram, личные рекомендации
- Получает только тёплые обращения — людей, которые уже поняли свой запрос

**Пользователь (потенциальный клиент):**
- Человек 25–55 лет с размытыми жалобами на здоровье
- Устал, плохо спит, не может похудеть, проблемы с ЖКТ или стресс
- Не знает с чего начать и боится «продажников»
- Ищет не диагноз — ищет понимание и первый конкретный шаг

---

### Как работает фильтрация (Kennedy: квалификация лида)

Приложение НЕ говорит «записывайся, это бесплатно, жми сюда». Оно предлагает пройти тест и разобраться — человек сам решает, нужно ли ему это. Те, кто дошли до финального CTA и нажали — это люди с реальной болью и реальной мотивацией. Остальные уходят сами.

---

### Метрики успеха ИКР

| Метрика | Цель |
|---------|------|
| Настройка под нового дистрибьютора | < 30 минут без технических знаний |
| Intro → CTA (доходимость воронки) | > 60% |
| CTA → реальный диалог с дистрибьютором | > 70% |
| Качество лида (человек с реальным запросом) | дистрибьютор оценивает субъективно — должен расти |

## Часть 2: Адаптированный промт для разработки

---

### Промт: Health Navigator Pro — White-Label Telegram Mini App

Мы хотим создать **универсальное Telegram Mini App** для специалистов по восстановлению здоровья и wellness-консультантов — работающее как **инструмент квалификации и прогрева**.

Приложение строится на базе существующего «Навигатора самочувствия» и наследует его архитектуру. Ключевое отличие — **полная white-label настройка через один файл** (`constants.ts`): любой дистрибьютор подставляет свои данные без единой строки кода.

---

#### Философия продукта (Kennedy + Brunson)

- **Фильтр, не продавец.** Приложение не продаёт продукты — оно помогает человеку осознать свой запрос. Те, кто дошли до CTA — уже квалифицированные лиды.
- **Один CTA.** Одно действие в конце — написать дистрибьютору. Никакого паралича выбора.
- **Живой человек в конце.** Никакой автоматики, никакого чат-бота. Только диалог со специалистом.
- **Никаких атрибутов компании.** Ни логотипов, ни названий MLM-бренда, ни цен на продукты. Только здоровье и симптомы.
- **Value first.** Сначала человек получает ценность (объяснение его состояния) — потом предложение.

---

#### Структура воронки (5 экранов)

```
intro → symptom → explanation → action → thankyou
```

**Экран 0: INTRO**
- Персонализация по имени из Telegram: «[Имя], за 2 минуты — узнай главную причину своего симптома»
- Социальное доказательство: счётчик прошедших + бейдж «Бесплатно»
- 3 буллет-выгоды (конкретика по Kennedy: время + результат + формат)
- Один CTA: «Пройти тест»
- OfferModal: разовый поп-ап — «Бесплатная 15-минутная консультация» (показывается один раз через 1.5 сек)

**Экран 1: SYMPTOM (Выбор симптома)**
- 6–8 тем — настраиваются дистрибьютором (из `TOPICS` в constants.ts)
- Примеры тем по умолчанию (подходят для большинства специалистов по здоровью):
  - Нет энергии / постоянная усталость
  - Плохой сон / просыпаюсь ночью
  - Тяжесть / проблемы с ЖКТ
  - Лишний вес / не могу похудеть
  - Стресс / раздражительность
  - Частые простуды / слабый иммунитет
  - Проблемы с кожей / волосами / ногтями
  - Хочу понять, с чего начать

**Экран 2: EXPLANATION (Объяснение)**
- Простым языком: что стоит за выбранным симптомом, почему это происходит
- Блок «На что стоит обратить внимание» — сетка 2×2 из ключевых систем организма
- Тёмный блок-мост: «Хочешь разобраться в своей конкретной ситуации?»
- Кнопка «Продолжить»

**Экран 3: ACTION (Карточка специалиста + CTA)**
- Фото, имя, роль («Специалист по восстановлению здоровья»)
- Ключевой факт: опыт работы, количество клиентов
- Карусель из 3–5 реальных отзывов с конкретными результатами (формат: до → после за N недель)
- Блок «Что будет на консультации»:
  1. Разберём главную причину твоих симптомов
  2. Составим план восстановления с конкретных шагов
  3. Ответим на твой главный вопрос
- Счётчик мест (`slotsLeft`, скрывается при 0)
- Один CTA: «Записаться на бесплатную консультацию»
- Подпись: «Без обязательств — просто разговор»

**Экран 4: THANKYOU**
- Подтверждение + 3 шага что будет дальше
- Кнопка «Поделиться навигатором» — вирусный рост

#### White-label: что настраивается в constants.ts

```typescript
// === ДАННЫЕ СПЕЦИАЛИСТА — менять только здесь ===

export const SPECIALIST = {
  name: 'Имя Фамилия',
  role: 'Специалист по восстановлению здоровья',
  photo: '/specialist.png',           // заменить фото в папке public/
  fact: '5 лет практики · 300+ клиентов · авторская программа восстановления',
  username: 'telegram_username',      // КРИТИЧНО: все CTA ведут сюда
  botUsername: 'my_bot_username',     // для кнопки «Поделиться»
  shareText: 'Прошёл(а) тест — точно описал мои симптомы. Попробуй:',
  slotsLeft: 3,                       // 0 = скрыть счётчик мест
};

// === ТЕМЫ СИМПТОМОВ — настроить под свою аудиторию ===

export const TOPICS: Topic[] = [
  {
    id: 'energy',
    title: 'Нет энергии',
    explanation: '...',               // объяснение простым языком
    focusAreas: ['...', '...', '...', '...'],  // 4 зоны внимания
  },
  // ... остальные темы
];

// === ОТЗЫВЫ — реальные кейсы с конкретными результатами ===

export const TESTIMONIALS = [
  {
    id: 'sleep',
    text: '«...»',
    author: 'Имя, возраст',

  },
  // ...
];

// === СОЦИАЛЬНОЕ ДОКАЗАТЕЛЬСТВО ===
export const SOCIAL_PROOF = {
  count: 0,       // обновлять реальной цифрой по мере накопления
  label: 'человек уже разобрались со своим состоянием',
};

// === ОФФЕР-МОДАЛ ===
export const OFFER = {
  title: 'Бесплатная 15-минутная консультация',
  subtitle: '...',
  bullets: ['...', '...', '...'],
  buttonText: 'Записаться',
  skipText: 'Пропустить',
  storageKey: 'offer_shown_v1',
};
```

#### Дизайн-система (наследуется из Навигатора самочувствия)

| Параметр | Значение | Настраивается? |
|----------|---------|---------------|
| Фон | `#f5f5f0` (тёплый офф-вайт) | Через `BG_COLOR` |
| Акцент | `#5A5A40` (оливковый) | Через `BRAND_COLOR` |
| Текст | `#1a1a1a` | Нет |
| Шрифт заголовков | serif | Нет |
| Шрифт текста | system sans-serif | Нет |

#### Технический стек (наследуется)

- React 19 + TypeScript + Vite + Tailwind CSS 4 + Motion
- Telegram WebApp SDK
- Деплой: Vercel (автодеплой из GitHub)
- Никакого бэкенда, никакой БД — чистый SPA

---

#### Чего намеренно НЕТ в приложении

| Что | Почему |
|-----|--------|
| Название компании (CoralClub и др.) | Любой MLM-бренд вызывает скептицизм раньше времени |
| Продукты с ценами | Это не каталог — это инструмент осознания |
| Бизнес-возможность / заработок | Другой инструмент, другая аудитория |
| Автоответы и чат-бот | Только живой диалог с дистрибьютором |
| Медицинские заявления | «Помогло мне», а не «лечит» |
| Несколько дистрибьюторов | MVP — один специалист, один бот |
